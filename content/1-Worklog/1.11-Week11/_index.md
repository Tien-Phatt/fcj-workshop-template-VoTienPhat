### Week 11 Objectives:

* Strengthen practical knowledge of AWS Identity and Access Management (IAM).
* Learn how to organize users and permissions using IAM Groups and IAM Roles.
* Understand role-based access control and secure authentication on AWS.
* Gain hands-on experience implementing IAM best practices for cloud resource management.

### Week 11 Achievements:

* Gained a comprehensive understanding of how IAM Groups simplify permission management for multiple AWS users.

* Successfully completed hands-on exercises involving:
  * Creating IAM Groups.
  * Creating IAM Users.
  * Assigning users to groups.
  * Validating user permissions.

* Understood the purpose and advantages of AWS IAM Roles for providing temporary credentials to users and AWS services.

* Learned the differences between:
  * IAM Users
  * IAM Groups
  * IAM Roles

* Successfully configured and tested **Switch Role** to securely access AWS resources using different permission sets.

* Applied the **Principle of Least Privilege** to design secure, scalable, and manageable access control policies.

* Consolidated knowledge from the AWS Security module, including identity management, access control, logging, and security best practices.

* Successfully completed the advanced IAM lessons and hands-on labs in the First Cloud Journey Bootcamp, providing a strong foundation for the remaining advanced AWS modules.
