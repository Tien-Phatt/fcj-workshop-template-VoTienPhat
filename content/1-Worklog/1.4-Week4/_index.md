### Week 4 Objectives:

* Understand the AWS Compute Services ecosystem.
* Learn how to deploy and manage Amazon EC2 instances.
* Explore the key components of Amazon EC2, including AMIs, Instance Types, EBS Volumes, and Security Groups.
* Gain hands-on experience launching and managing virtual servers on AWS.

### Week 4 Achievements:

* Gained a solid understanding of AWS Compute Services and their role in building scalable cloud applications.

* Learned the complete workflow for launching and configuring Amazon EC2 instances.

* Understood the purpose and functionality of:
  * Amazon Machine Images (AMIs)
  * Amazon Elastic Block Store (Amazon EBS)
  * Security Groups
  * EC2 Key Pairs

* Successfully launched and configured Amazon EC2 instances with appropriate networking and storage settings.

* Learned how to securely connect to Linux EC2 instances using SSH and manage the instance lifecycle.

* Understood the differences between the following EC2 operations:
  * Start
  * Stop
  * Reboot
  * Terminate

* Gained an introductory understanding of Elastic Load Balancing (ELB) and Amazon EC2 Auto Scaling for improving application availability and scalability.

* Successfully completed the Amazon EC2 lessons and hands-on labs in the First Cloud Journey Bootcamp.
